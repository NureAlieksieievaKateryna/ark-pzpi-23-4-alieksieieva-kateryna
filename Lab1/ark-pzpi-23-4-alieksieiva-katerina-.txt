ДОДАТОК Б
КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ (PROJECT VISION AND SCOPE)
1. КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ (CONCEPT AND PROJECT FRAMEWORK)
1.1 Концепція і рамки проєкту
Предметна область цього проєкту охоплює сферу надання та придбання підписок на послуги у закладах різних типів: кав’ярнях, фітнес-центрах, салонах краси, освітніх студіях, коворкінгах тощо. У сучасному світі все більшої популярності набуває модель регулярних платежів за послуги, коли клієнт отримує доступ до певного сервісу на певний період (тиждень, місяць, рік) за фіксовану плату.
Головна ідея полягає у створенні єдиної платформи, де клієнти можуть переглядати, купувати й продовжувати підписки різних закладів, а власники — створювати власні пропозиції, налаштовувати ціни, терміни дії та відстежувати відгуки користувачів.
Метою розробки є підвищення зручності взаємодії між клієнтами та закладами, автоматизація процесу придбання підписок, а також створення аналітичної бази, яка дозволить виявляти попит на певні послуги. Система надасть споживачеві єдину точку керування всіма його підписками, а бізнесу — новий стабільний канал продажів та аналітику попиту.
1.2 Бізнес-можливості
На ринку існують платформи для бронювання або онлайн-оплат послуг (наприклад, Booksy, LiqPay, Treatwell), проте більшість із них не надає можливості саме створення, продажу та управління підписками у форматі маркетплейсу.
Переваги програмного продукту:
•	Підтримка гнучкого конструктора підписок із різними строками, цінами та наборами послуг.
•	Централізований каталог, який збирає пропозиції від багатьох закладів.
•	Модуль зворотного зв’язку для аналізу потреб клієнтів.
•	Функція аналітики попиту на видалені або неактивні підписки (що дає закладам можливість відновити популярні пакети).
Модель монетизації:
1.	Комісія з кожної транзакції (5–10%).
2.	Преміум-доступ для закладів із розширеною аналітикою.
3.	Продаж анонімізованих даних про ринковий попит.
4.	Підписка для бізнесу на користування розширеним функціоналом.
Програмний продукт орієнтований на ринок малого та середнього бізнесу у сфері послуг. Завдяки зростанню популярності моделі підписок, проєкт є перспективним і має високий потенціал для швидкого масштабування.

2. КОНЦЕПЦІЯ РІШЕННЯ (VISION OF SOLUTION)
2.1 Окреслення концепції
Програмна система "Subscription Platform" є централізованим веб-застосунком (маркетплейсом), який слугує посередником між постачальниками послуг (закладами) та кінцевими споживачами. Метою створення є забезпечення користувачів зручним інструментом для пошуку, оформлення, продовження та управління підписками у різних закладах (фітнес-клуби, салони краси, коворкінги тощо). Система покликана автоматизувати процес купівлі послуг за підпискою та забезпечити бізнес не лише новими каналами продажів, але й унікальними ринковими даними, зводячи до мінімуму рутинні операції з обліку та стимулюючи зростання клієнтської бази за допомогою ексклюзивних пропозицій.
2.2 Головна функціональність
Функціональність, якою користуватимуться споживачі програмного продукту:
•	MF-1: Реєстрація користувача та авторизація через електронну пошту, телефон або соціальні мережі.
•	MF-2: Пошук закладів і підписок за категоріями, локацією, рейтингом або ціною із застосуванням фільтрів.
•	MF-3: Перегляд детальної інформації про підписку: опис послуг, вартість, термін дії, умови продовження, а також наявність ексклюзивних пакетів для користувачів платформи.
•	MF-4: Придбання підписки онлайн через інтегровані платіжні сервіси з миттєвим підтвердженням.
•	MF-5: Особистий кабінет користувача з історією покупок, поточними підписками та статусом оплат.
•	MF-6: Панель адміністратора для закладів: створення, редагування, видалення пропозицій, включаючи спеціальні ексклюзивні пакети для користувачів платформи, та перегляд базової аналітики продажів.
•	MF-7: Система рейтингів і відгуків користувачів із модерацією.
•	MF-8: Модуль нагадувань про закінчення підписки та рекомендації подібних послуг.
•	MF-9: Підтримка кількох мов інтерфейсу (локалізація).

3. РАМКИ ТА ОБМЕЖЕННЯ (SCOPE AND LIMITATIONS)
3.1 Рамки первинного випуску
Перша версія системи міститиме базовий функціонал (Minimum Viable Product, MVP), необхідний для повноцінного запуску:
Ключовий функціонал:
•	Реєстрація, вхід, створення профілю користувача.
•	Перегляд каталогу закладів та доступних підписок із базовим пошуком.
•	Можливість придбання підписок онлайн (одноразова оплата) та їх продовження.
•	Система відгуків і рейтингів.
•	Кабінет адміністратора для закладів із базовою аналітикою продажів (обсяг, кількість підписок).
Архітектурні та нефункціональні аспекти:
•	Адміністрування: Передбачено два рівні — супер-адміністратор системи (управління закладами, модерація) та адміністратори закладів (управління власними підписками).
•	Локалізація: Українська та англійська мови інтерфейсу.
•	Захист персональних даних: Базова підтримка безпеки (авторизація, HTTPS, шифрування даних) та відповідність вимогам GDPR та Закону України “Про захист персональних даних”.
•	Пристрої: Веб-версія, оптимізована для сучасних веб-браузерів (Chrome, Edge, Safari) та мобільних пристроїв (Responsive Web Design).
•	Бізнес-логіка: Реалізація процесів створення, редагування, пошуку та купівлі підписок, управління користувачами та обробки відгуків.
3.2 Рамки наступних випусків
У наступних версіях (після MVP) планується реалізація розширеної функціональності:
•	SR-1: Підписки з автоматичним поновленням через інтегровані платіжні сервіси.
•	SR-2: Система персональних рекомендацій на основі історії покупок, уподобань та геолокації користувачів.
•	SR-3: Розробка нативних мобільних додатків (iOS/Android) та інтеграція push-сповіщень.
•	SR-4: Розширена аналітика для закладів (порівняльні ринкові дані, heatmaps, статистика часу відвідувань, глибинний аналіз попиту), що є ключовою цінністю для залучення бізнесу.
•	SR-5: Підтримка програм лояльності, бонусних балів та промо-кодів, що стимулюють клієнтів використовувати платформу замість прямого каналу.
•	SR-6: Офлайн-режим збереження підписок у мобільному додатку (після його реалізації).

4. БІЗНЕС-КОНТЕКСТ (BUSINESS CONTEXT)
4.3 Operating Environment (Робоче середовище)
Цей розділ описує технології та вимоги до робочого середовища, необхідні для успішної реалізації та експлуатації системи.
Технологічний Стек:
•	Серверна частина (Backend): Java з фреймворком Spring Boot, обраний для забезпечення високої продуктивності, надійності та масштабування, особливо для критично важливих транзакційних систем.
•	База Даних (СУБД): PostgreSQL (для забезпечення цілісності та складності транзакцій).
•	Платіжні Інтеграції: API провідних платіжних систем (наприклад, LiqPay, Stripe).
•	Хостинг/Інфраструктура: Хмарні сервіси (наприклад, AWS, Google Cloud, Azure) для забезпечення масштабованості та доступності.
Основні Вимоги до Середовища:
•	Доступність: Система повинна працювати у режимі 24/7, з допустимим часом простою не більше 0.5 на місяць (включно з плановим обслуговуванням). Перебої в обслуговуванні допустимі лише під час планових оновлень, про які користувачі мають бути попереджені.
•	Надійність: Дані мають регулярно резервуватися (щоденно). Забезпечення цілісності транзакцій між системою та платіжними шлюзами.
•	Продуктивність: Час відгуку для критично важливих операцій (пошук, купівля підписки, вхід) не повинен перевищувати 2 секунди.
•	Цілісність: Всі дані користувачів та транзакцій повинні бути захищені за допомогою шифрування (SSL/TLS, шифрування даних у стані спокою). 
•	Географічне Розташування: Користувачі та заклади можуть бути широко розподілені географічно. Серверна інфраструктура має бути розташована таким чином, щоб забезпечити оптимальний доступ для цільового ринку.
