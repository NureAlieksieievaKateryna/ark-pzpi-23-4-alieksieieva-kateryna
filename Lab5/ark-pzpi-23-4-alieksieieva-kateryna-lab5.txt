3. ОПИС ВИКОНАНОЇ РОБОТИ
З метою спрощення процесу розгортання програмної системи, підвищення відтворюваності середовища виконання та зменшення залежності від конкретної платформи у даній лабораторній роботі було застосовано технологію контейнеризації Docker.
Серверна частина IoT-системи реалізована у вигляді застосунку на базі фреймворку Spring Boot. Для контейнеризації серверного застосунку було створено файл Dockerfile, який описує багатоступеневий процес побудови Docker-образу. На першому етапі використовується базовий образ із встановленим Maven та Java 17 для компіляції та збірки Spring Boot застосунку у виконуваний JAR-файл. На другому етапі застосовується полегшений образ з Java Runtime Environment, у який копіюється зібраний JAR-файл та здійснюється запуск серверного застосунку. Такий підхід дозволяє зменшити розмір фінального контейнера та підвищити безпеку розгортання.
Для оркестрації контейнерів використано файл docker-compose.yml версії 3.8, у якому описано взаємодію між окремими компонентами програмної системи. Було створено окремі контейнери для серверного Spring Boot IoT-бекенду, системи управління базами даних PostgreSQL, інструменту адміністрування PgAdmin, сервісу аутентифікації Keycloak, а також Swagger UI для візуалізації REST API.
Параметри підключення до бази даних передаються серверному застосунку через змінні середовища, що забезпечує гнучкість конфігурації та незалежність від конкретного середовища виконання. Взаємодія між контейнерами здійснюється через внутрішню Docker-мережу.
IoT-пристрій функціонує як зовнішній клієнт системи та не входить до контейнеризованого середовища. Він здійснює передачу даних сенсорів до серверної частини системи шляхом надсилання HTTP-запитів до REST API Spring Boot застосунку, що повністю відповідає концепції Internet of Things.
Для побудови та запуску програмної системи було використано команди docker compose build та docker compose up -d. Після запуску всі компоненти системи були успішно розгорнуті та готові до роботи, що підтверджено перевіркою доступності серверного API, бази даних та допоміжних сервісів.















4. ВИСНОВКИ
У ході виконання лабораторної роботи було виконано розгортання програмної системи на базі Spring Boot з використанням технології контейнеризації Docker. Налаштовано взаємодію між серверною частиною IoT-застосунку, базою даних PostgreSQL та допоміжними сервісами у середовищі Docker Compose. Проведено перевірку коректності роботи серверного застосунку та обміну даними з IoT-пристроєм через REST API. Отримані результати підтверджують доцільність використання Docker для забезпечення відтворюваності середовища, спрощення процесу розгортання та підвищення надійності роботи програмної системи
